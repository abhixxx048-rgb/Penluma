**This document teaches you three small but very common English mistakes: when to add an apostrophe (the `'` mark), when NOT to add one, and how to use "its" vs "it's" correctly. These mistakes appear in commit messages, PR titles, and Slack messages every day, and fixing them makes your writing look much more professional.**

**The main parts explained simply:**

- **Plain plurals — never use an apostrophe** — When you just mean "more than one thing," add `-s` or `-es` and nothing else. Write `PRs`, `APIs`, `bugs`, `tickets` — never `PR's`, `API's`, `bug's`. The apostrophe is not needed and is wrong here.

- **Possessives — use apostrophe + s** — When something *belongs to* a person or thing, add `'s`. Example: "the server's logs" means the logs belong to the server. "the team's decision" means the decision belongs to the team.

- **Plural possessives — apostrophe after the s** — When something belongs to *many* people or things (and the word already ends in `-s`), just add an apostrophe after: "the developers' commits" means commits belonging to several developers.

- **its vs it's — the unzip test** — `it's` (with apostrophe) always means "it is" or "it has." If you can replace it with "it is" and it still makes sense, use `it's`. If not, use `its` (no apostrophe). Example: "It's broken" = "It is broken" (correct). "Its config" = "It is config" (does not make sense, so use `its`).

- **Plurals spelling guide** — Words ending in `ch`, `sh`, `x`, `z`, or `s` add `-es` (branch → branches, class → classes). Words ending in consonant + `y` change to `-ies` (query → queries). Acronyms just get `-s` (API → APIs, ID → IDs).

- **Irregular plurals** — A few common tech words follow no rule: index → indexes, schema → schemas, datum → data. These are worth memorising separately.

- **Daily writing examples** — The document shows how these rules apply in commit messages, PR titles, Slack standups, code comments, and bug reports, so you can practise in real situations.

**What to do with this:**

Before you use an apostrophe, ask yourself: "Am I saying something *belongs to* someone?" If not, leave it out. And whenever you write "its" or "it's," try to expand it to "it is" — if that works, add the apostrophe; if not, leave it off.
