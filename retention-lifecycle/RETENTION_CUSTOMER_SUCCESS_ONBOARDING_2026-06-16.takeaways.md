**This document is about how a new print shop owner gets from "just signed up" to "my store is actually working and taking orders." It maps what the platform already does to help owners get started, what is missing, and a step-by-step plan to fix the gaps. Getting this right is critical because most businesses that leave a software product do so within the first 90 days - usually because they never reached a real "it works!" moment.**

**The main parts explained simply:**

- **Setup checklist** - A 5-step guide already built into the admin dashboard ("Upload logo," "Connect payment," etc.). It works, but its progress is stored only in the user's browser, so it resets on a different device and the platform owner can't see who is stuck.

- **Sample products and demo catalog** - When a store is created, some ready-made products are added automatically so the owner doesn't start with a blank store. This is a good foundation.

- **The "first order" moment** - Research shows that the real sign a new owner will stay is when they see their first real order arrive. The current checklist marks someone as "done" before they ever reach that moment. This needs to change.

- **Test order path** - Right now there is no way for an owner to try placing an order themselves before a real customer does. Adding this would let owners see the full experience and build confidence fast.

- **Server-side progress tracking** - Saving setup progress to the database instead of the browser. This makes progress visible to all staff on the account and lets the platform owner see who is on track and who is struggling.

- **Landlord visibility and nudge emails** - The platform owner (the SaaS) currently flies blind: they can't see which stores are active or stuck. Automated emails on Day 3 and Day 7 to stores that haven't finished setup would provide the minimum safety net.

- **In-app help and guidance** - There is no built-in help, no short walkthrough, and no plain-language explainers for things like "what is a payment gateway?" Non-technical owners need these to get unstuck on their own.

- **Tenant health score** - A simple score (Healthy / Stable / At-Risk) based on login activity, setup progress, and orders, used to spot struggling stores early before they leave.

- **Action Center integration** - An existing "things that need attention" feed on the dashboard. Unfinished setup can be wired into this so the reminder stays visible until the store is actually live.

**What to do with this:** Start by saving setup progress to the database (not just the browser) and adding a "first real order received" step to the checklist - these two changes unlock everything else. Then add a way for owners to place a test order themselves, so they can reach that confidence moment without waiting on a real customer.
