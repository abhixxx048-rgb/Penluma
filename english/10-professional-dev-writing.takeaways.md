**This document teaches you how to write like a senior developer - not by coding better, but by writing better. Every day you write commit messages, pull request descriptions, code review comments, Slack updates, and code comments. Small writing mistakes in these places make solid work look unpolished. Fix them once and your writing reads as professional immediately.**

**The main parts explained simply:**

- **Commit messages** - The one-line title of every save you make to the codebase. Always start with a command word ("Fix", "Add", "Update"), capitalize the first letter, and do not put a full stop at the end. Wrong: `fixed the bug.` Right: `Fix cart total when quantity is zero`
- **Pull request (PR) descriptions** - The summary you write when you ask a teammate to review your code. The title follows the same commit rules. The body answers three questions in full sentences: What did you change? Why? How can someone test it?
- **Code review comments** - Notes you leave on someone else's code. Never just say "wrong" or "no." Name the problem and suggest a fix. A question ("Could we guard this?") feels helpful; a command ("fix this") feels harsh.
- **Slack and standup messages** - Short daily updates about your work. Write full sentences with capital letters and full stops - not texting shorthand. Three sentences: yesterday, today, blockers.
- **Code comments** - Notes written inside the code itself. Always present tense, full sentence, capitalized. "Returns the cached price." - not "return cached price" or "this is returning…"
- **Much vs many** - A grammar rule that trips people up often. If you can count it (bugs, mistakes, tests), use **many**. If you cannot count it (time, work, research), use **much**.
- **Articles (a / an / the)** - Small words that English almost always needs before nouns. "Open **a** pull request" not "open pull request." When in doubt with a single countable thing, it needs *a*, *an*, or *the*.
- **Spelling watchlist** - Words commonly misspelled: *grammar* (not grammer), *research* (not reaseach), *separate*, *definitely*, *environment*.

**What to do with this:** Start with commit messages - use the formula `Fix/Add/Update + what + where`, no trailing period. Then move to Slack: write three full sentences for every standup, no `u`/`ur`/`pls`. These two habits alone make the biggest visible difference.
